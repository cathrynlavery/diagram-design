DP Security Matrix · Diagram Design · Parametric

Platform Access Matrix

Four canonical groups × eight platform components. Each cell holds a permission value (Admin / Full / R/W / Read / SELECT / Login / No access). One focal cell flags the diagram's central security claim — Data Consumers can only SELECT from the aggregated catalog, never raw or staging tiers.

Role grouping

4 canonical AD groups

  • Data Administrators — full platform control across every component
  • Data Engineers — R/W on storage + catalogs, drives the pipelines
  • Data Scientists — read-mostly on staging/aggregated, R/W in JupyterHub
  • Data Consumers — read-only on aggregated, no other reach

Cell vocabulary

5 visual levels

  • Full / Admin — administrative reach (ink bold, soft fill)
  • R/W — read + write (white fill, ink)
  • Read / SELECT — read-only or limited query (muted fill, muted text)
  • No access — paper-fill, soft text — explicit denial
  • Focal — accent border highlights the one critical access rule

Focal rule · One claim

Consumers × aggregated

  • Exactly one focal cell per diagram — the central security claim
  • Data Consumers can only SELECT from the aggregated catalog
  • 2-line cell content: primary value + sub explanation
  • Accent border (1.4) signals "audit me" without competing with the rest of the matrix
