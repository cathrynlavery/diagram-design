Medallion · Diagram Design · Parametric

Five-Tier Medallion Architecture

LFS quarterly survey through Raw → Anonymized → Staging → Aggregated → Archive. Each tier is a labelled bucket with its own tool, format, writer role, and concrete example payload. Promotion arcs over the top connect adjacent tiers and carry the transformation name (PII REMOVE, CLEAN+WEIGHT, AGGREGATE, LIFECYCLE) — arrow and label both clearly visible above the cards.

Tier organisation

Five quality / access levels

  • Raw — landing zone, PII intact, written by NiFi as it arrives
  • Anonymized — PII stripped, stable household IDs, partitioned Iceberg
  • Staging — weighted, cleaned, harmonised; data-science working zone
  • Aggregated — final indicators surface; the focal tier
  • Archive — cold tier, immutable, lifecycle-managed for retention

Promotion · Focal handoff

AGGREGATE arcs into the focal tier

  • Arcs route over the top of the tier strip — arrow + label both fully visible
  • Anchors at right/left shoulders (12 px inset), control points at y=0
  • Cubic peak around y=20; labels sit inside the arc at y=50, no mask needed
  • The AGGREGATE arc auto-promotes to accent because it lands in the focal tier

Write methods · Paths

SQL vs Notebook

  • SQL Path — Trino INSERT INTO … SELECT: set-based filter/join/aggregate
  • Notebook Path — DuckDB + Python/R in JupyterHub: row-iterative stats & ML
  • Both write paths apply between any pair of tiers; the choice is the engineer's
  • Path cards live below the tier strip — they describe how, not what
