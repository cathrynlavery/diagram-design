Current IT Landscape

The data pipeline before D4N: three phases (Collection, Processing, Dissemination) connected by manual file hand-offs — CSV pulls, email attachments, Excel copies. Two pain-points define the gap a platform proposal needs to close: a Shared Drive with no version control, and a DataZoa dissemination bottleneck. SQL Server (olive) survives the modernization.

Pain-points

  • Shared Drive. No version control; copies overwrite copies; the truth is whoever saved last.
  • DataZoa. One manual upload step between analysis and publication — the dissemination bottleneck.

Data exchanges

  • CSV / Email / Excel. Every flow into the Shared Drive is a manual file hand-off.
  • COPY. Analysts pull files locally for desktop tools; no audit trail.
  • CSV DL. Ministry partners pull statistics manually; no API.

Pre-platform tooling

  • SQL Server. On-premises core RDBMS — the survivor system the new platform federates.
  • Analyst Machines. SPSS / SAS / Stata / Excel; scripts live on individual workstations.
  • Civil Registry, Ministry Partners. External counterparties — dashed to mark scope boundary.
